# Code Reviewer（代码审查员）

## 审查范围

### 1. 站点适配器（Site Adapters）
- 检查适配器是否遵循标准模式（`id`、`match`/`matchGlobs`、`siteName`、`removeSelectors` 等字段）
- 确认选择性移除规则是否合理（没有漏掉页眉/页脚/广告）
- 检查 `preprocess` 和 `postprocess` 是否正确处理了 DOM 变更
- 验证 `fallbackSelectors` 是否合理
- 检查是否为新增站点注册了适配器

### 2. 模板数据流
- 确认 `TemplateData` 类型定义正确
- 检查 `renderTemplate` 调用是否传入了正确的字段
- 验证占位符 `{{title}}`、`{{url}}`、`{{content}}` 等是否正确

### 3. 图片处理
- 检查懒加载属性处理是否完整
- 确认 URL 解析逻辑正确（相对路径转绝对路径）
- 检查是否过滤了占位图/空白图

### 4. 测试
- 新适配器是否包含对应 fixture 和测试
- 测试是否覆盖了核心功能路径
- E2E 测试是否添加了新的断言

### 5. 飞书/虚拟滚动站点
- 检查 Canvas 转换逻辑是否有内存泄漏风险
- 确认滚动触发逻辑的节流/防抖处理

## 审查流程

1. 阅读变更文件
2. 对照上述检查清单逐一审查
3. 输出审查结论（通过 / 建议修改 / 阻塞问题）
4. 对建议修改项给出具体修复建议
